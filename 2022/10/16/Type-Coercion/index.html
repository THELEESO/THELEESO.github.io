Type Coercion

Type Coercion 強制型別轉換

在 JS 中，使用運算子或者 methos 都可以強制轉換其型別(type)。

'10' + 10 = '1010'
'10' - 10 = 0
'10' - 10 + 'Hi' = '0Hi'
'10' - 10 - 'Hi' = NaN

我們可以看到規則性：

1. 不管是原始型別是 String 還是 Number，只要再與 String 相加就會變成 String。
2. String(number) 與 Number 相減，就會自動轉換成數字。
3. 發生無法相減的情況下，就會產生 NaN(Not a Number)。

Math.methods()

使用 Math 相關 methods 都會產生 Type Coercion。

Math.abs('43') = 43
Math.floor('-43.123') = -43

toFixed()

有時候我們也會用 toFixed()來處理小數點，但需要明確知道他是會 return String。

10.123.toFixed(2) = `10.12`

快速轉換

除了一些 String 轉 Number 的 Methods(Number(), parseInt())，還有一種很簡單的方式就是使用 + 運算子。

+'123' = 123

其實這裡可以把它想像成，我們是要告訴 JavaScript 這是一個正數，所以強制在前面多了一個 + 。
所以當你使用 - 就會變成負數了！很簡單吧。

要注意的是，如果是要使用相加，必須在兩個運算子之間空一格。
相減的話，那就不需要 + 這個運算子囉。

100+ +'123' = 223

這個方式，可以讓你的程式碼看起來更乾淨，但如果不懂他的運作原理，反而會看不懂程式碼。
所以在表現 clean code 的同時，也要注重易讀性，有機會再來寫一下 clean code 的原則。
