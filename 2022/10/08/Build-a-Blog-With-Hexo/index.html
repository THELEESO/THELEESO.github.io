Building a Blog With Hexo

聊聊目的

一直以來都是靠 Notion 來做筆記，雖然功能強大，可是沒辦法完全自由化。所以決定來架個 Blog，想練練自己的能力。一開始是想使用 React.js+NEXT.js，但對目前的我來說有點麻煩，會花點時間，所以就先暫時放棄。

這篇將從 Github + Netlify + Porkbun 的簡化教學，會著重在比較麻煩的部分。

有關 HEXO 的使用方式可以到官方 Documentation。CLI 的部分，稍微懂一下 Git 的基礎知識和操作方式即可。不熟悉的話，可以看一下連猴子都懂的 Git。如果想要深入一點，可以去 Google 一下 Git Pro 2.0 。

Github

HEXO 建置完成後，你可以選擇上傳到 Github 或者 Netlify。最好的方式，就是先上傳 Github 再用 Netlify 來連結你的 repository。因為在 Github 上 Domain 的設定，會因為你每一次 upload 就要重新設定一次。

  1. 沒有帳號的，先創立一個帳號。
  2. 創立一個 new Repository 名稱是 Github 帳號.github.io 。
  3. 到 HEXO 中的 _confing.yml 找到 deploy。 沒有的話就在最後的 line 自行添加。
deploy:
  type: git
  repository: 放入剛剛創立的Repository網址
  branch: main (目前預設是 main，在以前會是 master)
  1. 在資料夾的目錄下，輸入 $ hexo d 進行 deploy 到 github 上。
  2. 到這裡便已經完成了 Github page 的部分。
  3. 如果沒有想要使用自己專屬的 Domain，那麼做到這裡即可。

Netlify

Netlify 是架設靜態網站的平台，個人是覺得十分好用。你如果是做前端的，也可以把作品直接放在這邊，他會給你官方預設的 Domain，便可以使用該 Domain 去連結你的網站。

為了解決每 git push 一次，就要重新設定 Domain，所以我用 Netlify。
假設你有其他託管的 server，也可以使用。Domain 的設定只設定一次，之後就算再怎樣發佈，也不會影響。

找到 Import new page，之後連結你的 Github 會出現下面的畫面

直接按 Deploy site 即可，不用多做設定，網路上很多都是舊版已失效了。舊版(master)會需要設定指令給 Netlify，新版(main)不需要做這些。如果跟著舊版做，網頁會跑不出來。


之後來到 Domain management 設定，請把你申請好的 Domain 貼上去讓他確認。
確認完他會給你四組類似 dns1.p04.nsnoe.net 這樣的 NameServers。把這四個複製下來，如果你不是像我一樣使用 Porkbun 的話，去你買 Domain 的網站，找到有可以設定 NS 的頁面貼上去即可。

由於 DNS 設定通常會是抓一天比較保守，簡單來說就是要先跑過全球的 server 來設定你這個 Domain 會連到你指定的頁面。所以設定好就等吧，不用急。

Porkbun

稍微介紹一下 Porkbun，看起來很像是個詐騙網站，但企業背景是很安全的，這點倒是不用太擔心你買的 Domain 被抄走。加上還附贈其他供應商需要額外加購的服務，我個人是覺得很超值。如果你對這供應商很擔憂，可以選擇 Google, Godaddy… 之類的。當然你也可以使用比價網 Domcomp，它會告訴你目前你指定的 Domain 哪裡最便宜。

接下來就是如何在 Porkbun 設定 Netlify 給你的四組 NS。
一樣找到 Domain management 的頁面。並點開你要使用的網域旁的 Details。展開的表格中找到 Authoritative Nameservers，把你那四組都丟進去，記得把裡面預設的 NS 都清空再貼近去。這裡就不附圖片了，有點懶。

他會給你一些警告，但不用理他，SAVE and DONE!
